# 前端框架说明

## 当前形态

当前前端是零依赖静态内部工具，运行在 `app/`：

```text
app/
  index.html
  styles.css
  src/
    app.js
    sample-data.js
    scoring.js
```

设计目标：

- 先把平台交互和业务流跑通。
- 不引入外部依赖，降低供应链风险。
- 保持模块边界清楚，后续可迁移到 React/Next.js。

## 页面模块

当前包含 7 个内部工具页面：

1. 总览：核心指标、今日优先处理、策略动作。
2. 机会池：商品机会排序、评分、利润、风险和供应链详情。
3. 上架审核：按审核状态展示上架草稿。
4. 订单履约：订单、SKU、金额、毛利、履约状态。
5. 供应商：采购价、MOQ、发货时效、评分、备选供应商。
6. 经营复盘：周级运营动作和分层指标。
7. 配置中心：评分权重、成本参数、平台佣金、风险规则。

## 代码边界

`app/src/scoring.js`

- 只放业务计算。
- 不依赖 DOM。
- 后续可以直接迁移到后端或共享包。

`app/src/sample-data.js`

- 当前本地示例数据。
- 后续替换为 API 返回数据。

`app/src/app.js`

- 页面状态、筛选、渲染和交互。
- 不新增复杂业务公式。

`app/styles.css`

- 统一视觉系统。
- 使用 restrained operational UI，适合内部运营工具。

## 后续迁移路线

### 1. 静态原型继续增强

适合确认交互：

- 增加导入弹窗。
- 增加上架草稿编辑表单。
- 增加订单详情抽屉。
- 增加配置编辑保存到 localStorage。

### 2. React/Next.js 版本

适合进入正式 MVP：

- 将每个视图拆成组件。
- 使用路由管理页面。
- 使用 API client 拉取后端数据。
- 使用表单库处理复杂编辑。

### 3. 后端接入

建议接口顺序：

1. `/api/opportunities`
2. `/api/supplier-products`
3. `/api/listing-drafts`
4. `/api/orders`
5. `/api/strategy/daily-report`
6. `/api/config/scoring`

## 视觉原则

- 信息密度优先，不做营销页。
- 卡片只用于具体对象，不嵌套卡片。
- 页面保持清晰的表格、看板、详情面板和指标卡。
- 高风险动作使用明显状态标记。
- 所有自动化动作都显示人工审核状态。
