# 开发路线

## 阶段 0：本地原型

目标：确认业务流程和评分逻辑。

交付：

- 静态工作台。
- 示例商品和供应商数据。
- 机会评分引擎。
- 推荐动作。
- 工程文档。

不做：

- 平台 API 对接。
- 登录系统。
- 自动采购。
- 自动上架。
- 真实订单同步。

## 阶段 1：MVP 内部工具

周期：2-4 周。

目标：让运营团队可以真实录入数据并完成半自动选品。

功能：

1. 用户登录和角色权限。
2. 商品机会池 CRUD。
3. 供应商库 CRUD。
4. 成本模板、佣金模板、物流模板。
5. 机会评分和解释。
6. 上架草稿生成。
7. 审核流。
8. CSV/Excel 导入导出。
9. 日报/周报。

建议技术：

- 前端：Next.js 或 React。
- 后端：FastAPI 或 NestJS。
- 数据库：PostgreSQL。
- 缓存/任务：Redis。
- 文件：本地对象存储或 S3 兼容存储。

部署建议：

- 个人试跑：本机 PostgreSQL Docker。
- 小团队共享：单台服务器 Docker Compose。
- 正式使用：应用服务器 + 托管 PostgreSQL。

验收：

- 运营可独立录入 100 个商品机会。
- 系统可自动排序并给出推荐动作。
- 每个机会有利润和风险解释。
- 所有发布/采购动作都有审核状态。

## 阶段 2：平台数据接入

周期：4-8 周。

目标：减少手动录入，逐步接入官方 API 或合规数据源。

功能：

1. Shopee 店铺授权。
2. TikTok Shop 店铺授权。
3. Ozon 店铺授权。
4. 商品、订单、刊登状态同步。
5. API 任务监控。
6. 同步失败重试。
7. API Key 加密管理。

基础设施要求：

- 后台任务队列。
- 平台授权表。
- 同步任务日志。
- 失败重试机制。
- 数据库每日备份。

注意：

- 优先使用官方开放平台 API。
- 不引入不可信爬虫插件。
- 如果 API 权限不足，使用平台导出的 CSV/Excel 作为过渡。

## 阶段 3：供应链增强

周期：4-8 周。

目标：从“找到商品”升级为“稳定履约”。

功能：

1. 1688 商品导入。
2. 供应商评分。
3. 备选供应商推荐。
4. 采购单。
5. 缺货预警。
6. 价格变动监控。
7. 物流方案比较。

验收：

- 每个上架商品至少有 1 个主供应商和 1 个备选供应商。
- 缺货或涨价时系统能提示替换。

## 阶段 4：订单履约

周期：6-10 周。

目标：订单进入系统后可追踪采购、物流和利润。

功能：

1. 订单聚合。
2. SKU 映射。
3. 采购单生成。
4. 物流单生成。
5. 订单状态同步。
6. 异常处理。
7. 售后和退款记录。

控制：

- MVP 后期可自动生成采购单，但付款仍需人工确认。
- 高风险订单必须人工处理。

## 阶段 5：策略自动化

周期：持续迭代。

目标：让系统每天给出经营动作，并允许低风险动作半自动执行。

功能：

1. 自动调价建议。
2. 广告预算建议。
3. 商品分层。
4. 国家扩展建议。
5. 海外仓备货建议。
6. A/B 测试记录。
7. 策略执行效果追踪。

自动化边界：

- 低风险调价可审批后批量执行。
- 上架、采购、下架仍建议保留确认机制。

## 阶段 6：完整版

目标：多平台、多国家、多供应商、多物流的跨境交易中枢。

能力：

- 官方 API 深度集成。
- 供应链履约闭环。
- 商品合规规则库。
- AI 本地化内容生成。
- 财务利润核算。
- 多角色协作。
- 完整审计日志。
- 自动策略回测。

基础设施形态：

- 托管 PostgreSQL。
- Redis。
- 对象存储。
- 日志系统。
- 监控告警。
- VPN 或零信任访问。
