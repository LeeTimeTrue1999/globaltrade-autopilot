# GlobalTrade Autopilot PRD

## 1. 背景

团队希望建立一个内部工具，用于自动化发现中国商品在东南亚和俄罗斯市场的交易机会，并支持从机会判断、国内供货匹配、海外上架、订单履约到日/周策略调整的经营闭环。

这个系统不应一开始追求完全自动化。跨境交易涉及平台规则、商品合规、物流、支付、汇率、知识产权和目的国监管，MVP 必须采用“系统推荐 + 人工审核 + 可追溯执行”的模式。

## 2. 产品定位

GlobalTrade Autopilot 是一个面向内部运营团队的跨境商品机会发现与交易工作台。

它回答五个问题：

1. 海外是否有人买？
2. 中国是否有稳定供货？
3. 扣除真实成本后是否仍有利润？
4. 商品是否能安全合规履约？
5. 今天和本周应该采取什么经营动作？

## 3. 用户角色

| 角色 | 目标 | 权限 |
| --- | --- | --- |
| 运营负责人 | 看机会、定策略、审批高风险动作 | 全量查看、审批、配置策略 |
| 选品运营 | 发现商品、维护机会池、发起上架 | 查看商品、编辑机会、提交审核 |
| 供应链运营 | 匹配供应商、维护采购价和履约能力 | 编辑供应商、采购和物流信息 |
| 店铺运营 | 维护平台刊登、价格、库存、订单 | 编辑刊登草稿、处理订单 |
| 财务/老板 | 查看利润、现金流、风险 | 只读报表和经营汇总 |
| 系统管理员 | 账号、权限、审计、集成配置 | 系统管理 |

## 4. 目标市场

MVP 目标区域：

- 东南亚：泰国、印尼、越南、菲律宾。
- 俄罗斯：作为独立风控区域处理。

MVP 目标平台：

- 东南亚：Shopee、TikTok Shop。
- 俄罗斯：Ozon。

后续平台：

- Lazada、Wildberries、Yandex Market、Shopify 独立站。

## 5. 核心原则

1. 先验证需求，再扩大铺货。
2. 只看价差不够，必须计算净毛利。
3. 俄罗斯方向必须单独做支付、物流、制裁和商品限制风控。
4. 默认半自动，所有高影响动作需要人工确认。
5. 所有推荐都要能解释：为什么推荐、数据来自哪里、风险是什么。
6. MVP 只做内部使用，不做公开 SaaS、多租户计费和营销页。

## 6. 业务流程

### 6.1 机会发现

输入：

- 海外平台候选商品。
- 平台价格、评论、评分、销量线索、排名。
- 搜索关键词和类目。
- 手动录入的市场观察。

处理：

- 计算需求分。
- 判断增长、稳定性和竞争强度。
- 标记低评分高销量、价格高但供货便宜、评论快速增长等机会。

输出：

- 商品机会池。
- 国家机会榜。
- 类目机会榜。
- 关键词机会榜。

### 6.2 供货匹配

输入：

- 海外商品标题、图片、规格、关键词。
- 1688/供应商库商品。
- 采购价、MOQ、发货时效、店铺信用。

处理：

- 文本匹配。
- 图片匹配，MVP 先保留接口占位。
- 多供应商聚合。
- 供应商评分。

输出：

- 推荐供应商。
- 备选供应商。
- 采购价区间。
- 履约风险。

### 6.3 利润测算

成本项：

- 国内采购价。
- 国内运输和包装。
- 国际物流。
- 平台佣金。
- 支付手续费。
- 税费。
- 广告预估成本。
- 汇率波动缓冲。
- 退货损耗。

输出：

- 推荐售价。
- 最低可接受售价。
- 预估毛利。
- 预估毛利率。
- 敏感性分析。

### 6.4 上架审核

系统生成：

- 当地语言标题。
- 卖点。
- 商品描述。
- 属性规格。
- 建议价格。
- 风险提示。

人工审核：

- 商品是否合规。
- 图片是否侵权。
- 标题是否含敏感词。
- 利润是否达标。
- 是否允许发布。

### 6.5 订单履约

MVP 先做订单导入和状态追踪，不直接对接支付。

状态：

- 待付款。
- 待采购。
- 已采购。
- 国内已发货。
- 国际运输中。
- 到达目的国。
- 派送中。
- 已签收。
- 异常。
- 已退款。

### 6.6 日/周复盘

日报：

- GMV。
- 订单数。
- 预估毛利。
- 异常订单。
- 需要审核的机会。
- 需要调价或下架的商品。

周报：

- 放大品。
- 观察品。
- 下架品。
- 换供应商品。
- 需要备货品。
- 国家和平台表现。

## 7. 功能需求

### 7.1 机会池

必须支持：

- 商品列表。
- 国家、平台、类目筛选。
- 机会分排序。
- 风险标记。
- 推荐动作。
- 详情查看。

### 7.2 评分引擎

评分维度：

| 维度 | 权重 | 说明 |
| --- | ---: | --- |
| 需求分 | 35% | 是否有人买、增长是否明显 |
| 毛利分 | 35% | 扣除成本后的利润空间 |
| 竞争分 | 15% | 竞争是否可进入 |
| 履约分 | 10% | 是否能稳定采购和发货 |
| 风险分 | -20% | 合规、IP、物流、平台政策风险 |

总分公式：

```text
OpportunityScore =
  DemandScore * 0.35 +
  MarginScore * 0.35 +
  CompetitionScore * 0.15 +
  FulfillmentScore * 0.10 -
  RiskScore * 0.20
```

动作规则：

| 条件 | 推荐动作 |
| --- | --- |
| 总分 >= 75 且毛利率 >= 25% 且风险分 < 35 | 小单测试 |
| 总分 >= 60 且毛利率 >= 18% | 观察/补数据 |
| 需求高但毛利低 | 找更低成本供应商 |
| 毛利高但需求低 | 暂不铺货 |
| 风险分 >= 60 | 人工合规复核 |
| 毛利率 < 12% | 不建议测试 |

### 7.3 策略中心

生成动作：

- 小单测试。
- 提价。
- 降价。
- 下架。
- 补充供应商。
- 切换物流。
- 加广告。
- 停止投放。
- 人工复核。

每个动作必须有：

- 触发原因。
- 影响范围。
- 预估收益或风险。
- 执行人。
- 审批状态。
- 执行结果。

### 7.4 配置中心

可配置：

- 目标国家。
- 目标平台。
- 汇率。
- 平台佣金。
- 支付手续费。
- 物流模板。
- 类目风险等级。
- 最低毛利率。
- 上架审核阈值。
- 自动策略阈值。

## 8. 非功能需求

### 8.0 项目运营要求

`docs/WORK_TODO.md` 是项目后续工作的统一运营文档和 TODO 看板。所有后续产品、设计、工程、数据、基础设施和文档工作都必须遵守：

- 开始工作前先阅读 `docs/WORK_TODO.md`，确认当前 TODO、决策、阻塞项和最近工作日志。
- 如果本次工作不在 TODO 看板中，先新增任务，再开始实现。
- 结束工作前更新任务状态、验证结果、决策变化、阻塞项和下一步。
- 涉及数据需求、服务器资源、采购项、架构变化、审批流程或运营平台/管理平台边界变化时，必须同步更新该文档。
- 该文档优先用于日常运营；`docs/TODO_DECISIONS.md` 只保留需要业务负责人拍板的问题。

### 8.1 可开发

- 模块清晰，评分引擎与界面分离。
- 所有规则配置化。
- 数据结构稳定，避免字段随意扩散。
- API 先按资源建模，后续可拆微服务。

### 8.2 可调整

- 国家、平台、物流、佣金、汇率不能写死。
- 评分权重必须可配置。
- 风险规则必须可追加。
- 商品类目和属性要支持平台差异。

### 8.3 安全可控

- MVP 不直接执行付款、采购和自动发布。
- 外部 API Key 加密保存。
- 所有高风险动作保留人工审批。
- 所有关键操作写审计日志。
- 禁止导入不可信脚本和浏览器插件。

### 8.4 可观测

- 任务执行状态可见。
- API 同步失败可重试。
- 每个推荐动作可解释。
- 报表口径固定。

## 9. MVP 验收标准

MVP 完成时应具备：

1. 可以录入或导入候选海外商品。
2. 可以录入或导入国内供应商商品。
3. 可以生成机会评分。
4. 可以看到推荐动作。
5. 可以生成上架草稿。
6. 可以查看日报/周报模拟输出。
7. 可以维护成本、佣金、汇率等配置。
8. 可以标记人工审核状态。
9. 所有动作不自动对外发布。

## 10. 暂定业务决策

以下内容先按默认值开发，后续可调整：

- 首发产品形态：内部 Web 工具。
- 首发国家：泰国、印尼、越南、菲律宾、俄罗斯。
- 首发平台：Shopee、TikTok Shop、Ozon。
- 首发供货源：1688 + 手动供应商库。
- 默认最低毛利率：25%。
- 默认小单测试数量：5-20 单。
- 默认所有发布/采购动作需要人工确认。
